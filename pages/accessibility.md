<section class="section" aria-label="Accessibility statement">
  <h1>Accessibility Statement</h1>
  <p class="section-lead">
    Faith Frontier Ecclesiastical Trust is committed to ensuring digital accessibility for people with disabilities. We continually work to improve the user experience for all visitors and apply relevant accessibility standards.
  </p>

  <h2>Conformance Status</h2>
  <p>
    This website aims to conform with the <strong>Web Content Accessibility Guidelines (WCAG) 2.1 Level AA</strong> and <strong>Section 508</strong> standards. These guidelines help make web content more accessible to people with disabilities, including those using assistive technologies such as screen readers.
  </p>

  <h2>Accessibility Features</h2>
  <p>This website includes the following accessibility features:</p>
  <ul>
    <li>Semantic HTML5 markup with proper heading hierarchy</li>
    <li>ARIA labels and landmarks for screen reader navigation</li>
    <li>Keyboard navigation support throughout the site</li>
    <li>Descriptive link text that conveys purpose and destination</li>
    <li>Alternative text for images where applicable</li>
    <li>Sufficient color contrast for readability</li>
    <li>Responsive design that adapts to various screen sizes and zoom levels</li>
    <li>Skip-to-content links for keyboard users</li>
  </ul>

  <h2>PDF Documents</h2>
  <p>
    Case filings and legal documents are often provided in PDF format as originally filed with courts. While we strive to include accessible versions where possible, some court-generated PDFs may not meet accessibility standards. When feasible, we provide plain-text summaries alongside PDF documents.
  </p>

  <h2>Ongoing Efforts</h2>
  <p>
    We regularly review and test this website to identify and address accessibility issues. This includes:
  </p>
  <ul>
    <li>Regular accessibility audits and testing with assistive technologies</li>
    <li>Adherence to semantic HTML and ARIA best practices</li>
    <li>Ongoing training and awareness of accessibility standards</li>
    <li>Incorporating user feedback to improve accessibility</li>
  </ul>

  <h2>Known Limitations</h2>
  <p>
    Despite our efforts, some content may not yet fully conform to accessibility standards. We are actively working to address any identified issues. Known limitations include:
  </p>
  <ul>
    <li>Some archival or historical documents may lack full accessibility features</li>
    <li>Third-party embedded content may not meet our accessibility standards</li>
  </ul>

  <h2>Compatibility</h2>
  <p>
    This website is designed to be compatible with:
  </p>
  <ul>
    <li>Recent versions of major browsers (Chrome, Firefox, Safari, Edge)</li>
    <li>Common screen readers (JAWS, NVDA, VoiceOver)</li>
    <li>Mobile devices and tablets</li>
    <li>Keyboard-only navigation</li>
  </ul>

  <h2>Technical Specifications</h2>
  <p>
    Accessibility of this website relies on the following technologies:
  </p>
  <ul>
    <li>HTML5</li>
    <li>CSS3</li>
    <li>WAI-ARIA</li>
    <li>JavaScript (progressive enhancement; core functionality available without JavaScript)</li>
  </ul>
